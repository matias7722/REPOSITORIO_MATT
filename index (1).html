Manual de Git & GitHub

⎇

Guía Rápida de GitHub

Domina el flujo de trabajo esencial: desde la configuración inicial hasta la colaboración en proyectos.

Configuración Inicial

Antes de empezar, debes identificarte en tu terminal. Esto vincula tus cambios a tu cuenta de GitHub.

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Conceptos Clave

  • Repositorio (Repo): La carpeta del proyecto que vive en la nube (GitHub).
  • Commit: Una "foto" o captura de tus archivos en un momento específico.
  • Branch (Rama): Una línea de tiempo paralela para trabajar en nuevas funciones sin romper el código principal.

Comandos Esenciales — Crear o Clonar

AcciónComandoDescripción
Clonargit clone [URL]Descarga un proyecto de GitHub a tu PC.
Inicializargit initConvierte una carpeta local en un repositorio de Git.

Ciclo de Cambios

Paso 01 Ver estado git status Revisa qué archivos han cambiado.
Paso 02 Preparar git add . Añade todos los cambios al área de preparación.
Paso 03 Confirmar git commit -m "..." Guarda los cambios localmente.
Paso 04 Enviar git push origin main Sube tus cambios a GitHub.

Ramas y Colaboración

Para trabajar de forma segura, se recomienda usar ramas:

  • Crear una rama: git branch nueva-funcion
  • Saltar a esa rama: git checkout nueva-funcion
  • Fusionar cambios: Una vez en main, usa git merge nueva-funcion
💡

Tip de Pro: Siempre usa git pull antes de empezar a trabajar para asegurarte de tener la versión más reciente del código que está en la nube.

El Pull Request (PR)

Este es el corazón de GitHub. Cuando terminas una mejora en tu rama y la subes (push), vas a la interfaz web de GitHub y verás el botón verde "Compare & pull request".

  1. Abre el PR: Describe qué hiciste.
  2. Revisión: Otros pueden comentar tu código.
  3. Merge: Una vez aprobado, se integra al proyecto principal.
